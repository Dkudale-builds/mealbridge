# Prototype Explanation

## Objective
The prototype demonstrates a working system for connecting food donors with recipients in a simple and efficient way.

---

## Features Implemented

- User interface for adding food donations  
- Display of available food items  
- Basic location-based filtering  
- Real-time data storage using Firebase  

---

## How the Prototype Works

1. User accesses the web application  
2. Donor enters food details through a form  
3. Data is saved in Firebase Firestore  
4. Other users can view available donations  
5. Matching is based on location data  

---

## Limitations

- No advanced matching algorithm  
- Limited user roles and permissions  
- No notification system  
- UI can be further improved  

---

## Assumptions

- Users provide correct location data  
- Internet connectivity is available  
- Firebase handles all backend operations  

---

## Future Scope

- Add authentication roles (donor, NGO, admin)  
- Improve UI/UX  
- Implement real-time notifications  
- Add mobile support
